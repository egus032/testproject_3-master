Привет, разработчик!

Чтобы быстро избавиться от необязательных файлов и зависимостей стандартного проекта, необходимо выполнить следующие действия

  1. Удалить папку /static/app/removeit
  2. Удалить зависимость bootstrap из src/main/frontend/bower.json. Блок overrides можно оставить как есть
  3. Удалить строку #7 (тема для bootstrap) из ftl/stylesheets.ftl
  4. Очистить содержимое файла ftl/header.ftl
  5. Очистить содержимое файла ftl/footer.ftl

Рекомендации

  • Каждая страница должна иметь одноимённую папку в static/app/ и файл в этой папке название_страниц.html и название_страницы.js
  • Верстальщикам, которые сомневаются в каком контроллере писать JS код, следует работать в файле static/js/frontend.js
  • Для реализации попапов следует пользоваться уже написанными средствами (сервисами и директивами модуля /static/app/common/mg-popup/mg-popup.js). Стилизовать попап можно в /static/app/mg-popup/mg-popup.css. Пример использования попапов можно посмотреть здесь

Особенности проекта

  • Все .js файлы и .css файлы поключаются автоматически в ftl/javascripts.ftl и ftl/stylesheets.ftl соответственно
  • При сборке проекта в профиле production все .js файлы собираются и минифицируются в один static/production.js. А так же для него создается source map
  • При сборке проекта в профиле production все .css файлы собираются в один static/css/production.css
  • При создании нового .js или .css файла следует пересобрать проект
  • Для сборки клиентской части проект используется Gulp
  • Для подтягивания зависимостей клиентской части используется Bower
